Karing 电脑端安装与使用教程

本文以 Windows 为例，MacOS 同理，也可参考。

一、下载安装 Karing

Windows 加速链接： 点击下载

MacOS 加速链接： 点击下载

Karing 是一个多平台网络代理软件，支持 iOS、Windows、Android、macOS 等平台。 其代码托管在 GitHub 上面，其他系统和架构可从 GitHub 上下载 Karing 软件安装包。

下载地址： https://github.com/KaringX/karing/releases

建议：优先使用 64 位 Windows 10 / 11 系统，直接下载最新版本的安装包即可。

二、安装与首次启动

1. 安装程序 / 解压

  1. 如果下载的是 .exe 安装包，双击运行安装向导，按提示下一步完成安装。
  2. 如果下载的是 .zip 压缩包：
    1. 右键压缩包，选择“全部解压”或“解压到当前文件夹”。
    2. 在解压后的文件夹中找到 karing.exe，双击即可运行。

2. 处理 Windows 安全提示

  1. 首次运行时，如果出现 “Windows 已保护你的电脑 / Microsoft Defender SmartScreen” 提示：
  2. 点击左下角的 “更多信息”，再点击 “仍要运行”。
  3. 如弹出 Windows 防火墙提示，勾选“专用网络”和“公用网络”，点击“允许访问”。

3. 语言与地区设置

  1. 第一次打开 Karing，会看到欢迎 / 协议页面，点击 Accept & Continue（同意并继续）。
  2. 在语言选择中选择 简体中文，界面会自动切换为中文。
  3. 在“国家 / 地区”页面选择你实际所在的地区，Karing 会根据地区应用默认分流策略。
  4. 如果有“新手模式”选项，保持开启即可，后面可以再慢慢研究高级规则。

三、Windows 桌面端界面速览

进入主界面后，大致会看到这些区域：

  • 顶部状态区：显示当前连接状态、实时速度、累计流量等。
  • 中间功能区：当前配置、规则模式（规则 / 全局）、节点延迟、日志等。
  • 左侧或底部导航：我的配置、服务器、设置 等入口。
  • 右下角托盘图标：Karing 最小化后会驻留在系统托盘，方便随时开关代理。
小提示：习惯后可以设置“开机自启”“最小化到托盘”等，方便日常使用。

四、添加订阅

要让 Karing 真正工作，需要导入一个订阅。

1. 在机场后台复制订阅链接

  1. 登录你的机场官网，进入“用户中心 / 我的订阅 / 仪表盘”。
  2. 找到 一键订阅 或 订阅地址，通常会有 Clash / Sing-box / 通用 等多种类型。
  3. 复制其中一个 Karing 支持的订阅链接（如 Clash / Sing-box / 通用订阅）。

2. 在 Karing 中导入订阅

  1. 在 Karing 主界面中，点击左侧的 “我的配置” 或顶部的“配置”入口。
  2. 点击 “添加配置”（有的版本是 “+” 按钮）。
  3. 在“配置链接 / 内容”输入框里粘贴刚才复制的订阅地址。
  4. 在“备注”里写上 Mugassn ,方便区分。。
  5. 确认更新频率（如每 6 小时 / 每天自动更新订阅）。
  6. 点击 保存 / 确定，等待 Karing 拉取配置。
  7. 回到“我的配置”列表，选中刚添加的这条配置，点击“更新”按钮手动刷新一次。
注意：如果 Karing 提示订阅地址无效，请先在浏览器中打开订阅链接，确认能正常访问。

五、选择节点并开启代理

1. 选择服务器节点

  1. 在主界面或左侧菜单中找到 “服务器” / “节点” 页面。
  2. 此处会列出当前订阅里的所有线路，如香港、日本、美国等。
  3. 可以点击“测速”按钮测试所有节点延迟，也可以只测当前可见节点。
  4. 根据延迟和地区选择一个节点，直接点击节点名称进行选中。
  5. 也可以使用 “自动选择” 功能，让 Karing 自动选取延迟最低的节点。

2. 打开电脑端代理开关

  1. 回到 Karing 主界面，找到醒目的总开关按钮（通常在顶部或中间）。
  2. 点击开关，将状态切换为“开启 / 已连接”。
  3. 第一次启动时，如果 Windows 再次弹出防火墙提示，点击“允许访问”。
  4. 当界面显示已连接、速度/流量开始变化时，说明代理已生效。

3. 测试网络是否生效

  1. 打开浏览器访问 https://www.google.com、https://www.youtube.com 等被墙网站。
  2. 如果可以正常打开，并且 Karing 内的实时速度有波动，表示配置成功。
切换规则模式：在规则模式 / 全局模式之间切换，可以根据需要决定是否所有流量都走代理。
